Continuous Deployment Commands

We have implemented an automated CI/CD pipeline for deploying our ah-repricer microservices through GitHub Actions. We frequently use the following commands for the deployment process.

Docker Commands

Repricer Service

Building Docker Image

>>> docker build -t drmaxv/repricer: .

Test Run Docker Image

>>> docker run -it drmaxv/repricer:

Push Docker Image

>>> docker push drmaxv/repricer:

Listing Service

Building Docker Image

>>> docker build -t drmaxv/listing: .

Test Run Docker Image

>>> docker run -it drmaxv/listing:

Push Docker Image

>>> docker push drmaxv/listing:

Feed Submission Service

Building Docker Image

>>> docker build -t drmaxv/feedsubmission: .

Test Run Docker Image

>>> docker run -it drmaxv/feedsubmission:

Push Docker Image

>>> docker push drmaxv/feedsubmission:

Default Service

Building Docker Image

>>> docker build -t drmaxv/default: .

Test Run Docker Image

>>> docker run -it drmaxv/default:

Push Docker Image

>>> docker push drmaxv/default:



Helm Commands

Installing Helm Charts

Repricer Chart

>>> helm install repricer apps/repricer/ah-core-engine --values apps/repricer/repricer/values.yaml -n ah-authentication

Listing Chart

>>> helm install listing apps/repricer/ah-core-engine --values apps/repricer/listing/values.yaml -n ah-authentication

Feed Submission Chart

>>> helm install feedsubmission apps/repricer/ah-core-engine --values apps/repricer/feedsubmission/values.yaml -n ah-authentication

Default Service Chart

>>> helm install default  apps/repricer/ah-core-engine --values apps/repricer/default/values.yaml -n ah-authentication



Upgrading Helm Charts

Repricer Chart

>>> helm upgrade repricer  apps/repricer/ah-core-engine --values apps/repricer/repricer/values.yaml -n ah-authentication

Listing Chart

>>> helm upgrade listing apps/repricer/ah-core-engine --values apps/repricer/listing/values.yaml -n ah-authentication

Feed Submission Chart

>>> helm upgrade feedsubmission apps/repricer/ah-core-engine --values apps/repricer/feedsubmission/values.yaml -n ah-authentication

Default Service Chart

>>> helm upgrade default  apps/repricer/ah-core-engine --values apps/repricer/default/values.yaml -n ah-authentication



Commands to retrieve and check resources against a namespace.

List Installed Charts

>>> helm list -n ah-authentication



Kubectl Commands

Scale Deployment

>>> kubectl scale deployment  --replicas=1 -n ah-authentication



Retrieve all resources Info

>>> kubectl get all -n  ah-authentication

Retrieve all Pods

>>> kubectl get pods -n ah-authentication

Retrieve all Deployments

>>> kubectl get deployments -n ah-authentication

Pod Logs

>>> kubectl logs -f  -n ah-authentication
